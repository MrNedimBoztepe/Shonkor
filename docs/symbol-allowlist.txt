# Backticked PascalCase names in the docs that are NOT Shonkor code symbols (#159).
#
# The guard (tests/Shonkor.Tests/DocsSymbolIntegrityTests.cs) fails on any backticked name that is
# neither declared in src/ nor listed here. It exists because arc42 described `PluginLoader` -- a
# DELETED class -- as compiling plugins at runtime, i.e. it advertised a removed RCE surface.
#
# ENFORCED RULES (#164). These are checked by the build, not left to your good intentions:
#   1. Every entry MUST sit under a [category] header. An entry outside a category fails.
#   2. Every entry MUST carry a "# reason" comment. A bare name fails.
#   3. A [deliberately-removed] entry MUST still be described as removed somewhere in the docs --
#      the guard requires the name to appear near "removed"/"deleted"/"no longer"/"has been".
#      Delete that prose and the entry stops being honest, so the build breaks.
#
# The point: you cannot turn a red build green by appending a name. Fix the doc instead.
